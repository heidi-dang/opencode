
### Resume Logic

The runtime's `tryResume()` method:
1. Scans `.opencode/runs/<run-id>/state.json`
2. Filters out terminal states (passed, failed, rolled_back)
3. Finds first non-terminal run
4. Maps status to stage:
   - "planning" → "planner"
   - "assigned"/"in_progress" → "dev"
   - "ready_for_reporter" → "havoc"
   - "gating" → "reporter"
5. Sets currentStage to the resumed stage
6. Continues execution from there

### No Duplicate Runs

Because the queue item is marked "in_progress" by planner, the resume logic won't create a duplicate run for the same task.
